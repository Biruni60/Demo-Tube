1.Scope of var, let, and const

i.Scope of var

Scope: var variables are function-scoped. This means they are accessible within the function in which they are declared, as well as within any nested functions.

ii.Scope of Let

Scope: let variables are block-scoped. This means they are confined to the block in which they are defined, such as a loop or an if statement. They are not accessible outside that block.

iii.scope of const

Scope: const variables are also block-scoped, just like let. They are confined to the block in which they are defined and are not accessible outside that block.

2.Use cases of null and undefined

Use of undefined

  • i.Default variable initialization: If you want to declare a variable and indicate that it doesn't yet have a value, JavaScript initializes it with undefined.
  • ii.Function parameters: If a function parameter is not provided with an argument when the function is called, it takes the value undefined.
  • iii.Implicit return: If a function doesn't explicitly return a value, it returns undefined by default.

Use of null

  • i.Explicitly signaling absence of value: If you want to indicate that a variable has no value or that an object property deliberately points to nothing, you can assign null.
  • ii.Resetting references: When you want to clear a reference to an object to release memory, you might set the reference to null.

3.REST API

A REST (Representational State Transfer) API is a common architectural style for designing networked applications, particularly web services. It is based on a set of principles that guide the design of APIs to ensure they are scalable, maintainable, and interoperable. REST APIs are widely used in web development for building client-server interactions, allowing clients (such as web browsers or mobile apps) to communicate with server-side resources.
